문서 / Task 작성 규칙

  • 각 Phase는 하나의 HTML 파일. 너무 커지면 sub-phase로 분할.
  • 한 Task는 목적 · 내용 · 코드 스니펫 · DoD(완료 기준) 네 영역을 가진다.
  • 코드 스니펫은 실제 빌드되는 형태를 지향. 헤더 누락·세미콜론 누락 없이.
  • 상태 뱃지: TODO / IN PROGRESS / DONE
  • Task 완료 시 본 인덱스의 Phase 진행 상태도 함께 갱신.
  • Phase가 끝나면 실제 구현과 차이 나는 부분을 회고로 남기고, 영향받는 다음 Phase TODO를 수정.
